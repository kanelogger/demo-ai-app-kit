# 基础配置模块需求

## 1. 模块目标

统一维护系统基础主数据，包括用户、部门、数据字典、角色和菜单。

## 1.1 导航归属

- 左侧菜单采用 `outlook bar` 风格
- 基础配置作为一级菜单组
- 个人信息、周报模板管理、用户信息维护、部门信息维护、数据字典维护、角色维护、菜单维护均在该一级菜单组下直接展示

## 2. 用户信息维护

- 字段：工号、密码、中文姓名、所属部门、直属领导、用户状态、角色集合
- 规则：
  - 工号唯一
  - 工号即登录账号
  - 直属领导为周报提报和审核流转依据

## 3. 部门信息维护

- 字段：部门编码、部门名称、状态
- 规则：不考虑多级部门

## 4. 数据字典维护

- 字段：字典类型编码、字典类型名称、字典项编码、字典项名称、排序号、状态
- 用途：为模板下拉字段提供数据来源

## 5. 角色维护

- 支持新增、编辑、启用/停用角色
- 支持查看角色下用户并进行分配

## 6. 菜单维护

- 字段：菜单名称、菜单路径、排序号、可访问角色
- 权限控制到菜单级别
- 超级管理员默认拥有全部菜单访问能力

## 7. Demo 要求

- 所有主数据维护均需支持模拟新增、编辑、状态切换
- 修改角色或菜单配置后，切换账号时需能体现权限变化
