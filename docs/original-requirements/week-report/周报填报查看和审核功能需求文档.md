# 周报填报查看和审核功能需求文档

## 1. 概述

周报填报查看和审核功能是系统的核心业务功能，涵盖了用户填报周报、查看自己填报的周报、领导审核下属周报以及查看权限用户浏览周报等完整的业务流程。

## 2. 功能模块

### 2.1 待我填报功能

#### 2.1.1 功能描述
拥有填报角色的用户可以查看需要自己填报的周报列表，并进行周报填报操作。

#### 2.1.2 业务规则
1. **填报时间限制**：周报只能在每周六、周日两天进行填报，其他时间不允许填报
2. **角色权限**：只有被分配了填报角色的用户才能看到对应的周报任务
3. **状态管理**：周报状态包括：未填报、已暂存、已提交、待审核、已审核

#### 2.1.3 功能要求

**查询和筛选功能：**
- **开始时间**：按周报开始时间筛选
- **结束时间**：按周报结束时间筛选  
- **周报名称**：按周报名称关键词搜索
- **状态筛选**：按填报状态筛选
- **分页显示**：支持分页查看，每页显示20条记录

**列表展示信息：**
- **周报名称**：显示周报模板名称
- **开始时间**：报告周期开始日期
- **结束时间**：报告周期结束日期
- **截止日期**：填报截止日期
- **填报状态**：当前填报状态（用不同颜色标识紧急程度）
- **操作按钮**：填报/编辑按钮

**状态标识规则：**
- 红色：距离截止日期不足1天
- 橙色：距离截止日期1-2天
- 灰色：距离截止日期超过2天

#### 2.1.4 填报界面设计

**填报标题信息（自动填充）：**
- **姓名**：当前登录用户姓名（不可编辑）
- **部门**：用户所属部门（不可编辑）
- **编制时间**：周报编制日期（可编辑，默认当前日期）
- **报告时间**：报告周期（不可编辑，显示开始时间-结束时间）

**填报正文内容：**
- 根据选择的周报模板自动加载相应的数据对象和数据项
- 按照模板配置的顺序展示不同的数据对象
- 每个数据对象以独立的表格或表单区域展示
- 根据数据项配置自动初始化相应的控件类型

**工具功能：**
1. **还原上周周报**：
   - 点击后系统自动导入上周同类型周报的内容
   - 用户可在导入内容基础上进行修改
   - 如果上周没有相应周报，提示用户

2. **暂存功能**：
   - 用户可以保存当前填写内容，下次继续编辑
   - 暂存状态的周报在列表中显示为"已暂存"
   - 用户可以多次暂存和修改

3. **提交功能**：
   - 验证所有必填项是否已填写
   - 验证数据格式是否正确
   - 提交后周报状态变为"待审核"
   - 提交后用户不能再修改内容

### 2.2 我填报的周报功能

#### 2.2.1 功能描述
用户可以查看自己填报的所有周报列表，包括查看周报内容和审核意见。

#### 2.2.2 功能要求

**查询和筛选功能：**
- **周报名称**：按周报名称关键词搜索
- **开始时间**：按报告周期开始时间筛选
- **结束时间**：按报告周期结束时间筛选
- **状态筛选**：按审核状态筛选
- **分页显示**：支持分页查看

**列表展示信息：**
- **周报名称**：显示周报模板名称
- **开始时间**：报告周期开始日期
- **结束时间**：报告周期结束日期
- **填报日期**：实际填报提交日期
- **审核状态**：待审核/已审核状态
- **审核意见**：直接在列表中显示领导的审核意见
- **操作按钮**：查看详情按钮

**详情查看功能：**
- 以只读形式展示周报的完整内容
- 显示审核意见和审核时间
- 显示审核人信息

### 2.3 待我审核功能

#### 2.3.1 功能描述
用户的直属领导可以查看需要自己审核的周报列表，并进行审核操作。

#### 2.3.2 业务规则
1. **审核权限**：根据用户维护的直属领导关系确定审核权限
2. **审核流程**：周报提交后自动分配给相应的直属领导审核
3. **审核结果**：审核完成后周报状态变为"已审核"

#### 2.3.3 功能要求

**查询和筛选功能：**
- **填报人**：按填报人姓名搜索
- **周报名称**：按周报名称关键词搜索
- **提交时间**：按周报提交时间筛选
- **部门筛选**：按填报人部门筛选
- **分页显示**：支持分页查看

**列表展示信息：**
- **填报人**：周报填报人姓名
- **部门**：填报人所属部门
- **周报名称**：周报模板名称
- **报告周期**：开始时间-结束时间
- **提交时间**：周报提交时间
- **操作按钮**：审核按钮

#### 2.3.4 审核界面设计

**周报内容展示：**
- 以表格化网页形式展示周报内容
- 不显示填报控件，直接显示填报内容
- 保持良好的排版和可读性
- 支持打印功能

**审核功能：**
1. **审核意见输入**：
   - 提供多行文本框供领导输入审核意见
   - 支持审核意见模板（如：优秀、良好、需改进等）
   - 审核意见为必填项

2. **审核提交**：
   - 点击提交审核按钮完成审核
   - 审核后周报状态变为"已审核"
   - 记录审核时间和审核人信息

### 2.4 周报查看功能

#### 2.4.1 功能描述
拥有查看角色的用户可以查看所有有权限查看的周报列表，但只能查看不能审核。

#### 2.4.2 功能要求

**查询和筛选功能：**
- **填报人**：按填报人姓名搜索
- **周报名称**：按周报名称关键词搜索
- **部门筛选**：按填报人部门筛选
- **时间范围**：按报告周期筛选
- **状态筛选**：按审核状态筛选
- **分页显示**：支持分页查看

**列表展示信息：**
- **填报人**：周报填报人姓名和部门
- **周报名称**：周报模板名称
- **报告周期**：开始时间-结束时间
- **填报时间**：实际填报时间
- **审核状态**：当前审核状态
- **审核人**：审核领导姓名
- **操作按钮**：查看详情按钮

**查看权限控制：**
- 根据用户的查看角色权限显示可查看的周报
- 不同角色可能看到不同范围的周报数据
- 系统自动过滤用户无权查看的周报

## 3. 周报状态流转

### 3.1 状态定义
1. **未填报**：周报任务已生成，但用户尚未开始填报
2. **已暂存**：用户已保存部分内容，但尚未提交
3. **已提交**：用户已提交周报，等待审核
4. **待审核**：周报已分配给直属领导，等待审核
5. **已审核**：领导已完成审核，流程结束

### 3.2 状态流转规则
```
未填报 → 已暂存 → 已提交 → 待审核 → 已审核
   ↑        ↓
   ←--------
```

- 用户可以在未填报和已暂存状态之间反复切换
- 一旦提交后，用户不能再修改内容
- 审核完成后，周报状态不再变更

### 3.3 时间限制规则
1. **填报时间窗口**：只能在周六、周日填报
2. **逾期处理**：超过截止时间未提交的周报标记为逾期
3. **审核时限**：领导应在收到审核任务后3个工作日内完成审核

## 4. 数据验证规则

### 4.1 填报验证
1. **必填项检查**：根据模板配置检查所有必填项
2. **数据格式验证**：验证数字、日期等特殊格式
3. **字符长度限制**：根据配置检查字符长度
4. **关联数据验证**：检查下拉选择项的有效性

### 4.2 提交验证
1. **完整性验证**：确保所有必填内容已填写
2. **时间验证**：检查当前时间是否在允许填报时间内
3. **权限验证**：确认用户有填报权限
4. **重复提交检查**：防止同一周报重复提交

## 5. 通知机制

### 5.1 填报提醒
1. **即将截止提醒**：在截止前1天发送提醒通知
2. **逾期提醒**：对未按时提交的用户发送逾期通知

### 5.2 审核提醒
1. **新增审核任务**：有新周报需要审核时通知领导
2. **审核逾期提醒**：超过3个工作日未审核时发送提醒

## 6. 界面设计要求

### 6.1 响应式设计
- 支持PC端和移动端访问
- 在不同屏幕尺寸下保持良好的用户体验
- 填报界面支持键盘操作和快捷键

### 6.2 用户体验
1. **操作引导**：为首次使用用户提供操作引导
2. **错误提示**：清晰的错误信息和解决建议
3. **自动保存**：填报过程中定期自动保存草稿
4. **操作确认**：重要操作（如提交、审核）需要确认

### 6.3 性能优化
1. **分页加载**：大数据量时采用分页加载
2. **缓存机制**：合理使用缓存提高响应速度
3. **异步操作**：非核心操作采用异步处理

## 7. 安全要求

### 7.1 权限控制
1. **角色权限**：严格按照角色权限控制功能访问
2. **数据权限**：用户只能访问授权范围内的数据
3. **操作权限**：不同状态下的操作权限控制

### 7.2 数据安全
1. **数据完整性**：确保周报数据不被篡改
2. **操作日志**：记录所有关键操作的日志
3. **备份恢复**：定期备份周报数据

## 8. 验收标准

1. 填报功能完整，支持所有模板配置的控件类型
2. 时间限制控制有效，非允许时间无法填报
3. 状态流转正确，权限控制准确
4. 审核功能完整，审核意见正确记录
5. 查看功能按权限正确显示数据
6. 通知机制正常工作
7. 界面美观易用，操作流畅
8. 数据验证规则生效，错误提示清晰
9. 性能满足要求，大数据量下响应正常
