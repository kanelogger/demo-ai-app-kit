# 首页模块需求

## 1. 模块目标

首页用于展示当前登录用户口径下的周报概览信息，包含统计卡片、系统公告、待办事项和分部门统计图表。

## 2. 页面结构

- 顶部 `top` 栏：显示系统名称、当前登录用户、演示业务日期、账号切换与退出操作
- 左侧导航栏：按菜单权限展示可访问菜单
- 中间内容区：采用多标签页工作台，标签支持切换和手工关闭
- 当前激活标签展示首页统计和待办内容

## 3. 展示内容

- 待填任务：当前自然周具备填报权限但尚未提交的模板数
- 已填任务：当前自然周状态为待审核或已审核的周报数量
- 总任务：当前自然周具备填报权限的模板总数
- 完成率：已填任务 / 总任务
- 系统公告：仅作为首页区块展示，不建设独立公告菜单
- 分部门统计：展示“本周应填总数 vs 本周已审核总数”
- 待办事项：展示当前账号相关的待审核任务
- 最近填报：展示当前账号最近的周报记录

## 4. 业务规则

- 统计口径按当前登录用户计算
- 草稿和已退回不计入已填任务
- 图表需展示所有部门，包括值为 0 的部门
- 首页中的待办和最近填报均采用模拟数据实时联动展示

## 5. Demo 要求

- 允许切换演示日期，验证周末填报、工作日只读规则
- 允许切换演示账号，体现不同角色看到的首页差异
- 风格保持与 `Demo-Ref` 参考原型一致，采用卡片、表格和清晰状态标签
