# 周报模板配置功能需求文档

## 1. 概述

周报模板配置功能是本系统的核心功能模块，允许管理员创建灵活的周报模板，并将模板分配给不同角色的用户。该功能支持类似Excel表格的灵活配置，能够满足不同部门和岗位的周报需求。

## 2. 功能模块

### 2.1 周报模板管理

#### 2.1.1 功能描述
管理员可以创建、编辑、删除周报模板，并查看所有已创建的模板列表。

#### 2.1.2 数据项要求
周报模板基本信息应包含：
- **模板编码**：唯一标识
- **模板名称**：周报模板的名称
- **模板描述**：模板用途和说明
- **模板类型**：个人周报、项目周报、部门周报等（关联数据字典）
- **创建人**：模板创建者
- **创建时间**：模板创建时间
- **模板状态**：启用/禁用/草稿状态
- **使用范围**：模板适用的部门或角色范围

#### 2.1.3 功能要求
1. **模板列表查看**：显示所有周报模板的列表，支持分页和搜索
2. **新增模板**：创建新的周报模板
3. **编辑模板**：修改已有模板的结构和配置
4. **删除模板**：删除不再使用的模板（需检查是否有关联数据）
5. **复制模板**：基于现有模板创建新模板
6. **模板预览**：预览模板的填报效果
7. **状态管理**：管理模板的启用/禁用状态

### 2.2 模板结构设计

#### 2.2.1 数据对象设计
每个周报模板可以包含多个数据对象，数据对象相互独立：

**数据对象属性：**
- **对象编码**：唯一标识
- **对象名称**：如"当前项目信息"、"售前项目"、"团队管理"等
- **对象描述**：对象用途说明
- **显示顺序**：在模板中的排列顺序
- **是否必填**：该对象是否为必填项
- **对象类型**：表格型、文本型、列表型等

#### 2.2.2 数据项设计
每个数据对象可以包含多个数据项：

**数据项属性：**
- **项目编码**：唯一标识
- **项目名称**：如"项目名称"、"项目状态"、"项目参与人员"等
- **项目描述**：项目说明
- **数据类型**：字符型、数字型、日期型
- **控件类型**：文本框、多行文本框、下拉选择框
- **是否必填**：该项是否为必填
- **默认值**：项目的默认值
- **验证规则**：数据验证规则
- **显示顺序**：在对象中的排列顺序
- **关联字典**：如果是下拉选择框，关联的数据字典项

#### 2.2.3 控件类型详细说明

**1. 文本框（TEXT）**
- 单行文本输入
- 支持最大长度限制
- 支持格式验证（如邮箱、电话等）

**2. 多行文本框（TEXTAREA）**
- 多行文本输入
- 支持行数设置
- 支持最大字符数限制

**3. 下拉选择框（SELECT）**
- 单选下拉选择
- 可配置选择某个数据字典作为选项来源
- 支持默认选中值

**4. 数字输入框（NUMBER）**
- 数字类型输入
- 支持最小值、最大值限制
- 支持小数位数设置

**5. 日期选择器（DATE）**
- 日期选择输入
- 支持日期格式设置
- 支持日期范围限制

### 2.3 模板角色分配

#### 2.3.1 功能描述
管理员可以为每个周报模板分配两类角色：填报角色和查看角色。

#### 2.3.2 角色类型
**1. 填报角色**
- 拥有该角色的用户需要填写该周报
- 可以对周报进行填写、修改、提交操作
- 在"待我填报"菜单中能看到该模板的周报任务

**2. 查看角色**
- 拥有该角色的用户可以查看该周报
- 只能查看周报内容，不能进行填报操作
- 在"周报查看"菜单中能看到该模板的周报列表

#### 2.3.3 分配规则
1. **多角色分配**：一个模板可以分配给多个填报角色和查看角色
2. **角色继承**：管理员角色默认拥有所有模板的查看权限
3. **动态更新**：角色分配变更后，用户权限实时生效
4. **权限检查**：系统根据用户角色动态显示可操作的模板

### 2.4 模板配置界面设计

#### 2.4.1 模板列表界面
- **功能区域**：新增模板、搜索、筛选、批量操作
- **列表展示**：模板名称、类型、状态、创建时间、操作按钮
- **操作按钮**：编辑、复制、预览、删除、角色分配

#### 2.4.2 模板编辑界面
- **基本信息区**：模板名称、描述、类型等基本信息
- **数据对象区**：数据对象的增删改，支持拖拽排序
- **数据项配置**：每个数据对象下数据项的详细配置
- **预览区域**：实时预览模板填报效果

#### 2.4.3 角色分配界面
- **填报角色区**：选择可以填报该模板的角色
- **查看角色区**：选择可以查看该模板的角色
- **用户预览**：显示当前配置下的影响用户列表

## 3. 模板配置示例

### 3.1 项目经理周报模板示例

**模板基本信息：**
- 模板名称：项目经理周报
- 模板类型：项目周报
- 模板描述：用于项目经理填报项目进展情况

**数据对象设计：**

**对象1：当前项目信息**
- 项目名称（文本框，必填）
- 项目状态（下拉选择框，关联"项目状态"字典，必填）
- 项目参与人员（多行文本框，必填）
- 项目进展（多行文本框，必填）
- 完成百分比（数字输入框，0-100，必填）

**对象2：售前项目**
- 潜在客户（文本框）
- 项目预算（数字输入框）
- 预计签单时间（日期选择器）
- 跟进状态（下拉选择框，关联"跟进状态"字典）

**对象3：团队管理**
- 团队规模（数字输入框）
- 本周团队动态（多行文本框）
- 人员变动情况（多行文本框）
- 培训计划（多行文本框）

### 3.2 个人周报模板示例

**模板基本信息：**
- 模板名称：个人周报
- 模板类型：个人周报
- 模板描述：用于普通员工填报个人工作情况

**数据对象设计：**

**对象1：本周工作总结**
- 主要工作内容（多行文本框，必填）
- 工作成果（多行文本框，必填）
- 遇到的问题（多行文本框）
- 工作时长（数字输入框，小时）

**对象2：下周工作计划**
- 计划工作内容（多行文本框，必填）
- 预期目标（多行文本框，必填）
- 需要的支持（多行文本框）

## 4. 模板使用流程

### 4.1 模板创建流程
1. 管理员进入"新增周报模板"页面
2. 填写模板基本信息
3. 创建数据对象，配置对象属性
4. 为每个数据对象添加数据项，设置项目属性
5. 配置控件类型和验证规则
6. 预览模板效果
7. 保存模板

### 4.2 角色分配流程
1. 管理员进入"周报模板列表"页面
2. 选择需要分配角色的模板
3. 点击"角色分配"按钮
4. 在填报角色区选择相关角色
5. 在查看角色区选择相关角色
6. 确认并保存分配结果

### 4.3 模板使用流程
1. 系统根据用户角色显示可操作的模板
2. 填报角色用户在"待我填报"中看到任务
3. 查看角色用户在"周报查看"中看到报表
4. 用户根据模板结构进行填报
5. 系统根据模板配置进行数据验证

## 5. 技术要求

### 5.1 数据存储
- 模板配置信息使用JSON格式存储
- 支持模板版本管理
- 模板变更时保持历史数据兼容性

### 5.2 性能要求
- 模板列表加载时间不超过3秒
- 模板编辑界面响应时间不超过1秒
- 支持大型模板（超过50个数据项）的流畅编辑

### 5.3 安全要求
- 只有管理员角色可以创建和编辑模板
- 模板删除需要权限验证
- 角色分配变更记录操作日志

## 6. 验收标准

1. 能够成功创建包含多个数据对象的复杂模板
2. 支持所有规定的控件类型和数据类型
3. 角色分配功能正常，权限控制有效
4. 模板预览功能准确反映填报效果
5. 数据字典关联功能正常工作
6. 模板复制功能完整可用
7. 界面操作流畅，用户体验良好
8. 模板配置的数据验证规则生效
