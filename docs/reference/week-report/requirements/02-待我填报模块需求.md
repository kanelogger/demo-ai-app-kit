# 待我填报模块需求

## 1. 模块目标

用于填报人查看自己当前自然周可填写的周报模板，并手工新增、继续编辑、暂存或提交周报。

## 2. 查询条件

- 周报名称
- 开始时间
- 结束时间

## 3. 列表字段

- 周报名称
- 周报周期开始时间
- 周报周期结束时间
- 当前状态
- 编制时间
- 操作

## 4. 操作规则

- 用户进入系统后，只要当前人员配置了某个周报模板的填报权限，列表中就必须出现一行待填报数据
- 用户在该行点击“填报”进入具体录入页面
- 不允许补报历史周报
- 周一至周五允许填写并暂存草稿
- 周六、周日允许继续编辑并正式提交
- 草稿和已退回允许继续编辑
- 待审核和已审核仅允许查看

## 5. 填报页面规则

- 从列表点击“填报/继续填报”后，在右侧工作区打开新标签页，不使用弹窗
- 标题区字段：姓名、部门、报告周期、直属领导
- 正文区按模板对象逐个展示
- 每个对象支持多行录入
- 每行字段按模板数据项渲染为单行文本、多行文本、下拉框或日期
- 支持“还原上周周报”

## 6. 暂存与提交

- 暂存为草稿，不强制要求直属领导和必填项完整
- 提交时必须校验：
  - 当前是否为周六或周日
  - 用户是否已配置直属领导
  - 模板必填项是否填写完整
- 提交成功后：
  - 状态变为待审核
  - 审核人固化为提交时的直属领导
  - 填报人不可继续修改
