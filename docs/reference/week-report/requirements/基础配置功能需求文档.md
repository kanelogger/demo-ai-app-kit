# 基础配置功能需求文档

## 1. 概述

本文档描述周报管理系统中基础配置功能的详细需求。基础配置功能是系统的核心支撑模块，包括用户管理、部门管理、数据字典、角色管理、菜单管理以及个人信息管理等功能。

## 2. 功能模块

### 2.1 用户信息维护功能

#### 2.1.1 功能描述
系统需要提供完整的用户信息维护功能，支持用户信息的增加、删除、修改和查询操作。

#### 2.1.2 数据项要求
用户信息应包含以下字段：
- **用户工号**：唯一标识，不可重复
- **用户登录名**：系统登录使用的用户名，不可重复
- **密码**：用户登录密码，需要加密存储
- **中文姓名**：用户的真实姓名
- **所属部门**：关联部门信息表
- **用户状态**：启用/禁用状态
- **直属领导用户名**：关联用户信息表，用于周报审核流程
- **用户角色**：一个用户可以拥有多个角色

#### 2.1.3 功能要求
1. **新增用户**：支持添加新用户，所有必填字段验证
2. **编辑用户**：支持修改用户信息，密码修改需要验证
3. **删除用户**：支持删除用户（逻辑删除）
4. **查询用户**：支持按工号、姓名、部门等条件查询
5. **密码加密**：用户密码必须加密存储，不得明文保存
6. **角色分配**：支持为用户分配多个角色
7. **领导关系维护**：支持设置用户的直属领导

#### 2.1.4 界面要求
- 用户列表页面：展示用户基本信息，支持分页和搜索
- 用户编辑页面：表单式编辑界面，支持角色多选
- 密码重置功能：管理员可重置用户密码

### 2.2 部门信息维护功能

#### 2.2.1 功能描述
系统需要提供部门信息的维护功能，采用单级结构设计。

#### 2.2.2 数据项要求
部门信息应包含以下字段：
- **部门编码**：唯一标识
- **部门名称**：部门的中文名称
- **部门描述**：部门职能描述
- **部门状态**：启用/禁用状态
- **部门负责人**：关联用户信息

#### 2.2.3 功能要求
1. **新增部门**：支持添加新部门
2. **编辑部门**：支持修改部门信息
3. **删除部门**：检查是否有用户关联，如有则不允许删除
4. **查询部门**：支持按部门名称等条件查询
5. **状态管理**：支持启用/禁用部门

### 2.3 数据字典维护功能

#### 2.3.1 功能描述
系统需要提供统一的数据字典维护功能，支持系统中所有需要配置的枚举值。

#### 2.3.2 数据项要求
数据字典应包含以下字段：
- **字典类型**：字典分类（如项目类型、周报类型等）
- **字典编码**：字典项的唯一编码
- **字典名称**：字典项的显示名称
- **字典值**：字典项的实际值
- **排序号**：显示顺序
- **状态**：启用/禁用状态
- **父级字典**：支持二级字典结构

#### 2.3.3 功能要求
1. **字典类型管理**：支持新增、编辑、删除字典类型
2. **字典项管理**：支持在字典类型下维护具体字典项
3. **层级支持**：支持二级字典结构
4. **排序功能**：支持字典项排序
5. **状态管理**：支持启用/禁用字典项

#### 2.3.4 预置字典类型
系统需要预置以下字典类型：
- 项目类型：研发项目、维护项目、测试项目等
- 周报类型：个人周报、项目周报、部门周报等
- 项目状态：规划中、进行中、已完成、已暂停等
- 用户状态：正常、禁用、离职等

### 2.4 角色维护功能

#### 2.4.1 功能描述
系统需要提供角色管理功能，支持角色的创建、编辑、删除和用户分配。

#### 2.4.2 数据项要求
角色信息应包含以下字段：
- **角色编码**：唯一标识
- **角色名称**：角色的中文名称
- **角色描述**：角色职能描述
- **角色状态**：启用/禁用状态
- **关联用户**：角色下的用户列表

#### 2.4.3 功能要求
1. **角色管理**：支持新增、编辑、删除角色
2. **用户分配**：在角色管理界面直接维护该角色下的用户
3. **权限控制**：一个角色可以分配给多个用户
4. **状态管理**：支持启用/禁用角色

#### 2.4.4 预置角色
系统需要预置以下角色：
- 系统管理员：拥有所有权限
- 部门经理：可审核下属周报，查看部门相关数据
- 项目经理：可填报项目周报，查看项目相关数据
- 普通员工：可填报个人周报
- HR管理员：可查看所有周报，管理用户信息

### 2.5 菜单维护功能

#### 2.5.1 功能描述
系统需要提供菜单管理功能，支持层级菜单结构和权限控制。

#### 2.5.2 数据项要求
菜单信息应包含以下字段：
- **菜单编码**：唯一标识
- **菜单名称**：显示名称
- **菜单图标**：FontAwesome图标类名
- **菜单链接**：相对地址路径
- **父级菜单**：支持多级菜单结构
- **排序号**：显示顺序
- **菜单状态**：启用/禁用状态
- **授权角色**：可访问该菜单的角色列表

#### 2.5.3 功能要求
1. **层级管理**：支持多级菜单结构
2. **链接配置**：维护菜单对应的功能链接地址
3. **权限配置**：为每个菜单配置可访问的角色
4. **图标管理**：支持为菜单配置图标
5. **排序功能**：支持菜单排序
6. **状态管理**：支持启用/禁用菜单

#### 2.5.4 菜单结构设计
```
├── 系统首页
├── 周报管理
│   ├── 待我填报
│   ├── 我填报的周报
│   ├── 待我审核
│   └── 周报查看
├── 模板管理
│   ├── 周报模板列表
│   └── 新增周报模板
├── 系统管理
│   ├── 用户管理
│   ├── 部门管理
│   ├── 数据字典
│   ├── 角色管理
│   └── 菜单管理
└── 系统设置
    ├── 系统公告
    └── 操作日志
```

### 2.6 个人信息查看功能

#### 2.6.1 功能描述
登录用户可以查看和修改自己的个人信息，特别是密码修改功能。

#### 2.6.2 功能要求
1. **信息查看**：用户可查看自己的基本信息
2. **密码修改**：用户可修改自己的登录密码
3. **信息更新**：用户可更新部分个人信息（不包括工号、角色等敏感信息）
4. **安全验证**：密码修改需要验证原密码

#### 2.6.3 界面要求
- 个人信息页面：显示用户基本信息
- 密码修改页面：原密码验证、新密码输入和确认

## 3. 非功能性需求

### 3.1 性能要求
- 用户列表查询响应时间不超过2秒
- 支持分页显示，每页默认显示20条记录
- 支持模糊查询和精确查询

### 3.2 安全要求
- 用户密码必须加密存储
- 支持密码复杂度验证
- 操作日志记录，追踪用户操作

### 3.3 易用性要求
- 界面简洁美观，操作直观
- 提供明确的操作提示和错误信息
- 支持键盘快捷操作

## 4. 接口要求

### 4.1 数据接口
- 与其他功能模块的数据关联接口
- 用户认证和授权接口
- 数据字典获取接口

### 4.2 外部接口
- 支持LDAP/AD用户同步（可选）
- 支持短信/邮件通知接口

## 5. 验收标准

1. 所有功能模块能够正常增删改查
2. 用户密码加密存储验证通过
3. 角色权限控制有效
4. 菜单层级结构正确显示
5. 数据字典功能完整可用
6. 个人信息修改功能正常
7. 界面美观，用户体验良好
